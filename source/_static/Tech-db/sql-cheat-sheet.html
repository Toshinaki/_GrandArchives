SQL

Cheat Sheet

 

SQL (Structured Query Language) is a domain-specific language used in programming and designed for querying a database.

Definitions:

  • SCHEMA
  • TABLE
  • VIEW
  • INDEX

Table Constraints
NOT NULL confirms that a column cannot store NULL value
UNIQUE ensures that each row for a column must have a different value
PRIMARY KEY a combination of NOT NULL and UNIQUE
ensures that the specific column or combination of two or more columns for a table have a unique identity which helps to find a particular record in a table more easily and quickly
CHECK ensures that the value stored in a column meets a specific condition
DEFAULT provides a default value when specified none for this column
FOREIGN KEY ensure the referential integrity of the data in one table to match values in another table;
The FOREIGN KEY CONSTRAINT is a column or list of columns which points to the PRIMARY KEY of another table.
The main purpose of FOREIGN KEY is, only those values will appear which are present in the primary key table. 
  • CREATE TABLE <table_name>( 
    column1    data_type[(size)] ,  
    column2    data_type[(size)] ,  
    constraint(constraint_name) 
    FOREIGN KEY  [column1,column2...] 
    REFERENCES [primary_key_table] (column_list_of_primary_key_table) ...);
  • CREATE TABLE <table_name>( 
    column1    data_type[(size)] ,  
    column2    data_type[(size)]
    REFERENCES <primary_key_table> [column_list_of_primary_key_table] ...);

Operators
Arithmetic Operators
+ - * / % add, subtract, multiply, divide or find remainder on numeric values
Comparison Operators
=; >; <; >=; <=; <>; find out if equal to, greater than, less than, greater than equal to, less than equal to, or not euqal to on any compatible data types
IS NULL; IS NOT NULL testing if a column is null
In SQL, a comparison between a NULL value and any other value (including another NULL) using a comparison operator will result in a NULL, which is considered as false for the purposes of a where clause
Logical Operators
AND OR NOT Logical comparison
IN checks a value within a set of values separated by commas and retrieve the rows from the table which are matching;
Operates on any set of values of the same datatype
BETWEEN tests an expression against a range. The range consists of a beginning, followed by an AND keyword and an end expression;
Operates on numeric, characters, or datetime values
ANY compares a value to each value in a list or results from a query and evaluates to true if the result of an inner query contains at least one row;
Operates on a value to a list or a single - columns set of values
ALL compares a value to every value in a list or results from a query.
must be preceded by the comparison operators and evaluates to TRUE if the query returns no rows;
Operates on a value to a list or a single - columns set of values
SOME compare a value to each value in a list or results from a query and evaluate to true if the result of an inner query contains at least one row;
Operates on a value to a list or a single - columns set of values
EXISTS tests whether a subquery fetches at least one row. When no data is returned then this operator returns FALSE;
Operates on table
LIKE Operator
% represent a sequence of 0 (zero) or more characters
_ represents a single character
[charlist] represents any single character within a charlist
[^charlist] or [!charlist] represents any single character other than the charlist

Aggregate Functions
COUNT() 
COUNT(*)
COUNT( [ALL|DISTINCT] expression )
returns the number of rows in a table satisfying the criteria specified in the WHERE clause
sets the number of rows or non NULL column values
returns 0 if there were no matching rows.
DISTINCT eliminates the repetitive appearance of the same data
ALL counts only NON NULL value
* counts the NULL value also
SUM() 
SUM ([ALL | DISTINCT] expression )
returns the SUM of all selected columns
DISTINCT returns the SUM of unique values
ALL applies to all values
AVG() 
AVG ([ALL | DISTINCT] expression )
returns the average of all non NULL values
DISTINCT returns the SUM of unique values
ALL applies to all values
MAX() finds the maximum value or highest value of a certain column or expression
MIN() finds the minimum value or lowest value of a column or expression

DDL
SCHEMA
CREATE 
CREATE SCHEMA [schema_name] [AUTHORIZATION owner_name]
[DEFAULT CHARACTER SET char_set_name]
[PATH schema_name[, ...]]
[ ANSI CREATE statements [...] ]
[ ANSI GRANT statements [...] ];
  • schema_name:
    The name of a schema to be created. If this is omitted, the user_name is used as the schema name.
  • AUTHORIZATION owner_name:
    Identifies the user who is the owner of the schema. If not mentioned the current user is set as the owner.
  • DEFAULT CHARACTER SET char_set_name:
    Specify the default character set, used for all objects created in the schema.
  • PATH schema_name[, ...]:
    An optional file path and file name.
  • ANSI CREATE statements [...]:
    Contains one or more CREATE statements.
  • ANSI GRANT statements [...]:
    Contains one or more GRANT statements.
ALTER 
ALTER SCHEMA schema_name [RENAME TO new_schema_name] [OWNER TO new_user_name]
  • schema_name:
    The name of an existing schema.
  • new_schema_name:
    The new name of the schema.
  • new_owner:
    The new owner of the schema.
DROP 
DROP SCHEMA <schema name>

TABLE
CREATE 
CREATE TABLE <table_name>(
    column1    data_type[(size)]    constraint,
    column2    data_type[(size)]    constraint,
    ...
    constraint);
  • table_name:
    Name of the table where data is stored.
  • column1,column2:
    Name of the columns of a table.
  • data_type:
    Char, varchar, integer, decimal, date and more.
  • size:
    Maximum length of the column of a table.
ALTER 
  1. ALTER TABLE <table_name>(
    ADD column1    data_type[(size)]    constraint,
    ADD column2    data_type[(size)]    constraint,
    ...);
  2. ALTER TABLE <table_name> 
    DROP(column1);
  3. ALTER TABLE <table_name> 
    MODIFY (column1 data_type[(size)] constraint);
  4. ALTER TABLE <table_name> 
    ADD CONSTRAINT constraint_name constraint_type(column1);
  5. ALTER TABLE <table_name> 
    DROP CONSTRAINT constraint_name;
  6. ALTER TABLE <table_name> RENAME TO <new_table_name>
  7. ALTER TABLE <table_name> RENAME <column_name> TO <new_column_name>
  1. add column
  2. remove column
  3. change datatype, size or constraint
  4. add constraint to column
  5. remove constraint
  6. rename a table
  7. rename a column
DROP 
DROP TABLE [table name];
 drops the table and all the relationships related to the tables;
A table which has a FOREIGN KEY and referenced by another table can not be dropped and if required to drop, the foreign key constraint or the referencing table should be dropped first.
  • table_name:
    Name of the table
TRUNCATE 
TRUNCATE TABLE [table name];
 remove all data in a table
VIEW
CREATE 
CREATE VIEW view_name [(column_names)]
    AS select_statement
CREATE 
CREATE VIEW view_name [(column_names)]
    AS select_statement
    WITH [CASCADED | LOCAL] CHECK OPTION
 WITH CHECK OPTION is used only on views that allow updates to their base tables;
ensure that only those data which may be read by the view can be inserted, updated, or deleted through the view.
CASCADED performs the check option for the current view and all views upon which it is built
LOCAL performs the check option only for the current view, even when it is built upon other views.

    AS select_statement

    select_statement

DML
TABLE
INSERT 
INSERT INTO <table name> [(col1,col2,col3...col n)]
    VALUES (value1,value2,value3...value n)[, (value1,value2,value3...value n), ...];
  • table_name:
    Name of the table where data will be inserted.
  • col1,col2,col3,col n:
    Column of the table. Optional.
  • value1,value2,value3,value n:
    Values against each column;
    If column names are not provided, insert values into columns by order;
    can insert multiple rows at once. 
INSERT INTO <table name> [(col1,col2,col3...col n)]
    select_statement
 Inserting the result of a query in another table
UPDATE 
UPDATE <table name> 
    SET <column1>=<value1>,<column2>=<value2>,...
    [WHERE <condition>];
  • table_name:
    Name of the table to be updated.
  • column1,column2:
    Name of the columns of the table.
  • value1,value2:
    New values.
  • condition:
    Condition(s) using various functions and operators. Optional.

DELETE FROM { table_name | ONLY (table_name) }
    [{ WHERE search_condition | WHERE CURRENT OF cursor_name }]
  • table_name:
    Name of the table.
  • search_condition:
    specify a search condition (logical expression) that has one or more conditions.

DQL
SELECT 
SELECT * | {[ALL | DISTINCT] column | expression [AS alias]...}
FROM <table_name>
[WHERE <condition>]
[GROUP BY <columns>]
[HAVING <search_condition>]
[ORDER BY <columns> [ASC | DESC]];
[LIMIT <n> [OFFSET <offset>]]
  • *:
    Specifies all the columns of a table.
  • DISTINCT:
    Eliminate the repetitive appearance of same data.
  • column:
    Column names.
  • table_name:
    Name of the table.
  • WHERE <condition>:
    Filter unwanted rows from the result.
  • GROUP BY <columns>:
    The GROUP BY clause is used with the SELECT statement to make a group of rows based on the values of a specific column or expression.
    The SQL AGGREGATE function can be used to get summary information for every group and these are applied to an individual group.
    The WHERE clause is used to retrieve rows based on a certain condition, but it can not be applied to grouped result; use HAVING instead
  • HAVING <search_condition>:
    Specifies a search condition for a group or an aggregate.
    HAVING is usually used in a GROUP BY clause, but even if GROUP BY clause is not used, HAVING can be used to function like a WHERE clause
  • ORDER BY <columns>:
    Sorts the result of a query according to the values in one or more specific columns
  • LIMIT <n> [OFFSET <offset>]:
    skip offset rows and return the next n rows
JOIN
combines rows from two or more tables if they have at least one common field and have a relationship between them
CROSS JOIN explicit 
SELECT * 
FROM table1 
CROSS JOIN table2;
or implicit 
SELECT *
FROM table1, table2
  • returns Cartesian product of rows from tables
  • by using a WHERE clause, CROSS JOIN produces the equivalent of an INNER JOIN
INNER JOIN EQUI JOIN uses only quality (=) as join condition (using other comparison operators makes a NON EQUI JOIN) 
SELECT * 
FROM table1 [INNER] JOIN table2 
ON table1.column_name = table2.column_name;
or 
SELECT * 
FROM table1, table2 
WHERE table1.column_name = table2.column_name;
or if columns used in an equi-join have the same name 
SELECT * 
FROM table1 [INNER] JOIN table2 USING (column_name)
NATURAL JOIN 
SELECT *
FROM table1
NATURAL JOIN table2;
  • columns with the same name of associated tables will appear once only
  • do not need ON clause
OUTER JOIN LEFT OUTER JOIN 
SELECT * 
FROM table1 
LEFT [OUTER] JOIN table2 ON table1.column_name = table2.column_name;
returns all matching rows plus rows from the frist table that do not match any row in the second table
for unmatching rows, values of columns from second table will be NULL
RIGHT OUTER JOIN 
SELECT * 
FROM table1 
RIGHT [OUTER] JOIN table2 ON table1.column_name = table2.column_name;
returns all matching rows plus rows from the second table that do not match any row in the first table
for unmatching rows, values of columns from first table will be NULL
FULL OUTER JOIN 
SELECT * 
FROM table1 
FULL OUTER JOIN table2 ON table1.column_name = table2.column_name;
returns all matching rows plus rows that do not match
for unmatching rows, columns without values will be filled with NULL
self join (unary) 
SELECT a.column_name, b.column_name... 
FROM table1 a, table1 b 
WHERE a.common_filed = b.common_field;
 join a table to itself, especially when the table has a FOREIGN KEY which references its own PRIMARY KEY
UNION 
SELECT column_name(s) FROM table1
UNION [ALL]
SELECT column_name(s) FROM table2;
  • Each SELECT statement within UNION must have the same number of columns
  • The columns must also have similar data types
  • The columns in each SELECT statement must also be in the same order
by default, UNION returns only distinct values; ALL allows duplicate values too
EXCEPT 
SELECT column_name(s) FROM table1
EXCEPT
SELECT column_name(s) FROM table2;
 returns distinct values in the first SELECT that not in the second SELECT
INTERSECT 
SELECT column_name(s) FROM table1
INTERSECT
SELECT column_name(s) FROM table2;
 returns distinct values in both the first SELECT and the second SELECT
